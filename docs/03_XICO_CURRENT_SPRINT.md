# Current Sprint

## Current focus
- refine Archivo
- preserve premium editorial feel
- improve repeat-opening mechanics
- continue seals system
- define the next step for the companion
- keep product decisions tied to habit, not just aesthetics

## Current design truth
The product already has strong editorial taste.
The risk now is not ugliness.
The risk is building something admired once and not reopened.

## Current visual caution
Avoid:
- generic profile-screen patterns
- mascots that feel childish
- over-loud color blocks that weaken sophistication
- redesigns that create sameness instead of hierarchy

## Current companion direction
Preferred direction:
- Alebrije logic for the emotional companion layer
- Glifo logic for seals and symbolic achievements

Execution rule:
The companion must never feel like a children’s app mascot.

## Current product question
For every feature, ask:
Does this help someone return tomorrow?
Does this create identity, ritual, or memory?
Or is it only visually impressive?

## Current open decisions
- exact role of companion on surface screens vs Archivo
- deterministic alebrije system vs later illustrated system
- daily habit loop design
- seal hierarchy and visual differentiation
- how to sharpen Archivo into something users want to revisit and share

## Current delivery preference
Default to:
- small, shippable steps
- exact implementation guidance
- concrete tradeoffs
- honest critique
- no decorative over-explanation
