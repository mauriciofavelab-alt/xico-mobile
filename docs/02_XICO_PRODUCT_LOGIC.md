# XICO Product Logic

## Core architecture
XICO has four pillars:
- Índice
- Cultura
- México Ahora
- Archivo

## Role of each pillar

### Índice
The cover.
The editorial front door.
The place where the monthly issue, strongest stories, and narrative hierarchy are felt.

### Cultura
The deep cultural archive.
Exhibitions, cinema, literature, photography, arts, visual intelligence, slower relevance.

### México Ahora
The live layer.
Gastronomy, mezcal, trends, current conversation, what is happening this week, what feels alive now.

### Archivo
The private emotional layer.
Saved pieces, seals, profile, identity, memory, personal cultural record.

Archivo must not feel like a generic account tab.
It should feel like a personal passport, dossier, or editorial self-portrait.

## Monthly issue logic
Each month should have a theme.
That theme should unify:
- selected stories
- curation
- events
- mood
- Ruta
- editorial framing

The monthly issue gives cohesion and reason to return.

## Seals
Seals must feel earned, culturally meaningful, and identity-building.

They should not feel like:
- generic gamification
- empty achievement badges
- childish trophies

They should feel closer to:
- recognition
- symbolic progress
- cultural literacy
- personal editorial identity

Glifo logic is strong for seals because it adds weight, sophistication, and system value.

## Companion logic
A companion is acceptable if it increases emotional attachment and return behavior without infantilizing the product.

Directionally:
- Alebrije logic works well for the emotional companion layer
- Glifo logic works better for seals and symbolic achievement systems

The companion must feel:
- ownable
- culturally grounded
- visually distinct
- non-cartoonish
- premium enough to coexist with high-editorial design

## Personalization
Personalization should not flatten the editorial point of view.
It should sharpen relevance.

The product should preserve strong editorial judgment while adapting:
- ordering
- emphasis
- ritual prompts
- saved value
- return cues

## Product warning
Do not mistake personalization for generic feed logic.
Do not turn XICO into a tasteless recommendation engine.

## Retention principle
The product must create reasons to return daily or weekly, not just admire once.

Habit value should come from:
- editorial rhythm
- identity-building
- saved/archive accumulation
- meaningful progression
- culturally specific relevance

## Design principle
A feature is not justified only because it looks premium.
A feature is justified if it strengthens:
- clarity
- identity
- ritual
- memory
- return behavior
